porcupine — The top level Porcupine package¶

Doing import porcupine gives you access to the things documented here. It may work for other submodules too, but it’s recommended to be more specific when importing them, like from porcupine import tabs.

Basic Stuff¶

If you haven’t read the plugin writing introduction yet I recommend reading it first. Then you’ll have a much better idea about what these functions do.

porcupine.get_main_window()[source]¶

Return the tkinter root window that Porcupine is using.

porcupine.get_tab_manager()[source]¶

Return the porcupine.tabs.TabManager widget in the main window.

Running Porcupine from Python¶

When Porcupine is started normally, it’s roughly equivalent to running a script like this in Python:

import porcupine
from porcupine import pluginloader

porcupine.init()
pluginloader.load(pluginloader.find_plugins())
porcupine.run()

This is useful for implementing plugins that need to start a new Porcupine process.

See porcupine.pluginloader documentation for more about loading plugins. Note that the plugins assume that porcupine.init() has been called, so they must be loaded after calling init().

porcupine.init(verbose_logging=False)[source]¶

Get everything ready for running Porcupine.

The verbose_logging option corresponds to the --verbose argument. Run porcu --help for more information about it.

porcupine.get_init_kwargs()[source]¶

Return a dictionary of the keyword arguments that were passed to init().

This is useful for invoking init() again in exactly the same way.

porcupine.run()[source]¶
porcupine.quit()[source]¶

Calling this function is equivalent to clicking the X button in the corner of the main window.

First the can_be_closed() method of each tab is called. If all tabs can be closed, a <<PorcupineQuit>> virtual event is generated on the main window, all tabs are closed with close_tab() and widgets are destroyed.

Version Information¶

porcupine.version_info¶
porcupine.__version__¶

These variables contains information about the current Porcupine version. version_info is a tuple like (0, 28, 1), and __version__ is a string such as '0.28.1'. Use version_info for checking if Porcupine supports a specific feature:

if porcupine.version_info >= (0, 28):
    # use some feature that was added in Porcupine 0.28
else:
    # display an error message or do things without the new feature
